## Input 태그
- name이 Key값이다. (물론 id= “” 를 써줄 수 있지만, 쓰지 말자)
*태그의 name 값이 키(Key)*로 해서 값(Value)가 전송된다.
즉, *request*에 값이 전달될 때 Map과 마찬가지로 *Key와 Value 쌍의 형식으로 데이터가 저장*된다.
Id 써도 되긴 하는데 중복이 되면 안되어서…그냥 name을 더 많이 씀. 
- input 3총사 : type, name, value
```
<input type = "text" name = "userid" value = "test"/>
```
- input태그에서 던져주는 값들은 대부분 String 자료형태로 넘어온다. 안 정했다 싶으면 모두 String으로 주고 받는다.

## Button Tag
 - button은 그 자체로 type=“submit”이다.  Submit은 서버로 보내버린다.
 - type을 빼도 submit 된다. 
 - <should> type=“button”으로 지정줘야한다. 
 - 소위 말해, 이벤트 핸들러가 작동하는 태그이다.
 - 다시 말하지만, 이벤트는 시스템에 다 정의되어있다. 

## Form Tag
- form 태그 안으로 덮지 않으면 서버로 정보를 보낼 수 없다. (Html)
- attribute { ‘action’, ‘method’} 이건 html에 정해진 속성이다. 
```
$("#register_form").attr({
    'action': '/registerimpl',
    'method': 'post'
});
```
